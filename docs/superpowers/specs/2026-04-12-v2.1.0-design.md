# RuSwitcher v2.1.0 Design Spec

## Overview

Three changes ship in 2.1.0:

1. **Bugfix: word boundary counting** тАФ structural keys (space, Enter, Tab, arrows, Backspace) were skipped when `event.flags` contained a stale modifier (e.g. `.maskAlternate`), causing the word-length counter to span across word boundaries and convert too many characters.
2. **Per-App Layout Memory** тАФ remembers which keyboard layout was active in each application and restores it on app switch.
3. **Install & Restart** тАФ one-click update from the "new version available" dialog: downloads DMG, verifies sha256, replaces `.app`, restarts.

## 1. Bugfix: Word Boundary Counting

**File:** `KeyboardMonitor.swift`, `handleKeyDown(keyCode:flags:)`

**Root cause:** The modifier filter (`if !modifiers.isEmpty { return }`) ran before structural key checks (space, Enter, Tab, arrows, Backspace). If a space event arrived with a stale `.maskAlternate` flag, the space was completely ignored, so `currentWordLength` kept incrementing across word boundaries.

**Fix:** Move structural key handling (space, Enter, Tab, arrows, Backspace) above the modifier filter. Only letter counting requires the modifier check.

**Already implemented** in current working tree.

## 2. Per-App Layout Memory

### Storage

In-memory dictionary `[String: String]` mapping `bundleIdentifier` to `layoutID`. Not persisted тАФ resets on app restart (layouts reset anyway).

### Mechanism

New class: `PerAppLayoutManager`

- Subscribes to `NSWorkspace.didActivateApplicationNotification`
- On notification:
  1. Save current layout ID for the **previous** app's `bundleIdentifier`
  2. Look up stored layout for the **new** app's `bundleIdentifier`
  3. If found and different from current: switch layout via `LayoutSwitcher`, update status icon
  4. Reset KeyboardMonitor word counters (typing context lost on app switch)
- Tracks `previousBundleID` to know which app we're leaving

### Integration

- `AppDelegate` creates and owns `PerAppLayoutManager`
- Calls `start()` / `stop()` based on `SettingsManager.shared.perAppLayout`
- On setting change: start or stop immediately
- After layout restore: call `appDelegate.updateStatusIcon()`

### Setting

- Key: `com.ruswitcher.perAppLayout`
- Type: `Bool`, default: `false`
- UI: Checkbox in General tab of Settings window, below the auto-switch checkbox
- Localized label + hint for all 12 languages

## 3. Install & Restart

### Trigger

New button in the existing update-available alert: "Install & Restart" alongside existing "Download" / "Skip" / "Later".

### Flow

1. User clicks "Install & Restart"
2. Show progress (indeterminate тАФ DMG is small, ~1 MB)
3. Download DMG from the release URL to `/tmp/RuSwitcher-update.dmg`
4. Verify sha256 of downloaded DMG against `sha256` field from `version.json` on GitHub
5. Mount DMG via `hdiutil attach -nobrowse -readonly`
6. Locate `.app` inside mounted volume
7. Replace current app bundle atomically via `FileManager.default.replaceItemAt(_:withItemAt:)`
8. Unmount DMG via `hdiutil detach`
9. Clean up temp file
10. Restart via existing `restartApp()` method in AppDelegate

### Verification

- Add `"sha256"` field to `version.json` (computed from the final notarized+stapled DMG)
- `UpdateChecker` fetches `version.json`, extracts both `version` and `sha256`
- After download, compute sha256 of the local DMG file and compare
- On mismatch: show error alert, do not install

### Error Handling

- Download failure: show alert with error description
- sha256 mismatch: show alert "Download verification failed"
- Replace failure (permissions, SIP): show alert suggesting manual install, open DMG in Finder as fallback
- All errors are non-fatal тАФ user can always fall back to manual download

### Code Location

Extend existing `UpdateChecker.swift` with:
- `installAndRestart()` async method
- Private helpers: `downloadDMG()`, `verifySHA256()`, `mountAndReplace()`, `cleanup()`
